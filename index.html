E-mail

E-mail que receberá os alertas gerados pelo monitor.

Deve ser o mesmo Gmail usado para gerar a senha de app salva no GitHub como EMAIL_SENHA.

Periodicidade padrão

A periodicidade efetiva do agendamento é definida no workflow do GitHub Actions. Este campo fica registrado no JSON para controle do painel.

Tipos de alerta

Novos artigos e edições
Correspondências de palavras-chave
Chamadas de artigos (call for papers)

Integração com GitHub

Use esta seção para salvar a configuração diretamente no repositório. O token fica apenas nesta aba do navegador e não é gravado no arquivo.

Para salvar: Contents: Read and write. Para salvar e rodar agora: inclua também Actions: Read and write.

Fluxo recomendado

Depois de alterar revistas, palavras-chave ou e-mail, clique em Salvar configuração no GitHub. Use Salvar e rodar monitor agora apenas quando quiser testar imediatamente. A execução automática continuará acontecendo pelo GitHub Actions.

Fonte e filtros

O leitor usa sempre o journals_config.json publicado no GitHub Pages. Ao clicar em “Atualizar leitor”, a configuração mais recente é carregada automaticamente e os feeds RSS são consultados. Alguns publishers podem bloquear leitura direta no navegador; nesses casos o monitor por e-mail continua funcionando normalmente via GitHub Actions.

Clique em “Atualizar leitor” para carregar a configuração publicada e consultar os feeds.
